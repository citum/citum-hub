# csl26-o18j Fix Plan (2026-02-27)

## Scope

Bean: `csl26-o18j`  
Title: "Fix missing components in bibliography"  
Target symptom: migrated bibliography templates omit `containerTitle` and/or `doi` components that exist in CSL 1.0 output.

## Current Baseline (as of 2026-02-27)

Recent oracle aggregate runs do **not** show the original missing-component pattern:

- `node scripts/oracle-batch-aggregate.js styles-legacy/ --top 10`
  - Bibliography 100%: `7/10`
  - Top component issue: `publisher:extra` (no `containerTitle:missing`, no `doi:missing`)
- `node scripts/oracle-batch-aggregate.js styles-legacy/ --top 50`
  - Tested 17 styles, Bibliography 100%: `13/17`
  - Top component issue: `publisher:extra` (no `containerTitle:missing`, no `doi:missing`)

Conclusion: `csl26-o18j` appears stale or already fixed by adjacent template-compiler work; plan is verification-first, then targeted repair only if reproducible.

## Execution Plan

1. Reproduce-or-close gate
- Run a wider oracle/component scan and explicitly grep for:
  - `containerTitle:missing`
  - `doi:missing`
- If both counts are zero, mark `csl26-o18j` completed as "no longer reproducible."
- If non-zero, continue with steps 2-5.

2. Isolate pipeline stage
- Add temporary diagnostics in `crates/citum-migrate/src/main.rs` around:
  - post-`compile_bibliography_with_types`
  - post-deduplicate/reorder/grouping passes
- Determine whether components are lost in:
  - `TemplateCompiler::collect_occurrences` / `merge_occurrences`, or
  - post-compile pass chain in `passes/*`.

3. Fix strategy (minimal and local)
- If compiler-stage loss:
  - patch `crates/citum-migrate/src/template_compiler/mod.rs` to preserve `TitleType::ParentSerial` and `SimpleVariable::Doi` through grouping/merge.
- If pass-stage loss:
  - patch the specific pass (`deduplicate`, `reorder`, or `grouping`) to avoid suppressing/removing these components incorrectly.
- Keep changes narrowly scoped to bibliography path; avoid citation behavior changes.

4. Regression tests
- Add compiler-level test(s) in `crates/citum-migrate/src/template_compiler/mod.rs` (or a dedicated migrate test file) that assert:
  - `container-title` source maps to `title: parent-serial` in bibliography template when present.
  - `DOI` source maps to `variable: doi` and is retained after post-processing.
- Prefer fixture patterns that include condition branches and list/group nesting.

5. Verify end-to-end
- Re-run targeted oracle on failing styles first.
- Re-run batch aggregate to confirm:
  - missing-component counters are zero
  - no new regressions in ordering/duplicate metrics.

## Acceptance Criteria

- `containerTitle:missing == 0` and `doi:missing == 0` in batch oracle aggregate output.
- New/updated tests fail before the fix and pass after the fix.
- No regression in citation scores for top-parent set.

## Risk Notes

- Over-aggressive deduplication/suppress logic can hide valid fields in type-specific branches.
- Reorder/grouping passes can relocate components and make them look "missing" in component parser output if punctuation/structure changes unexpectedly.

## Exit Conditions

- If no reproducible `containerTitle`/`doi` missing cases are found in current baseline runs, close `csl26-o18j` and spin out residual bibliography mismatch work under the active issue (`publisher:extra`) rather than forcing unnecessary code changes.
